Обґрунтування вибору типу бази даних

Аналіз вимог до системи

Система управління бібліотекою повинна забезпечувати:

 збереження інформації про книги
 збереження інформації про авторів
 облік читачів
 контроль видачі та повернення книг
 пошук книг за різними параметрами
 формування звітності

Дані мають чітку структуру та взаємозв’язки між сутностями:
 Книга → Автор
 Книга → Категорія
 Читач → Видача
 Книга → Видача

Це свідчить про наявність реляційної структури даних.

Вибір типу бази даних

Для реалізації системи було обрано **реляційну базу даних (SQL)**.

Причини вибору:

1. Чітка структурованість даних.
2. Наявність зв’язків між таблицями.
3. Підтримка транзакцій.
4. Забезпечення цілісності даних (PRIMARY KEY, FOREIGN KEY).
5. Можливість складних запитів (JOIN, GROUP BY).


Обрана СУБД

Для реалізації можна використовувати одну з наступних систем:

 MySQL
 PostgreSQL
 SQLite

Для навчального або невеликого проєкту доцільно використати **MySQL або PostgreSQL**, оскільки вони:

 підтримують багатокористувацький режим
 мають високу продуктивність
 є безкоштовними
 широко використовуються у реальних проєктах


Висновок

Реляційна база даних є оптимальним рішенням для системи управління бібліотекою, оскільки забезпечує:

 структурованість
 надійність
 контроль цілісності
 можливість масштабування
 підтримку транзакцій

Обраний тип бази даних повністю відповідає функціональним та нефункціональним вимогам системи.
